# 番茄录入草稿 — 快速启动

## 一键启动

双击项目根目录的 `启动番茄录入.bat`，或在命令行运行：

```bash
python -m FanqieUploader.playwright_batch
```

## 前提条件

1. **关闭所有 Chrome 窗口** — 脚本需要复制登录态到临时目录，Chrome 运行时会锁定 User Data
2. **番茄后台已登录** — 首次运行时在弹出的 Chrome 窗口中手动登录番茄，之后登录态会保存在临时目录，后续运行无需重复登录
3. **Novel/ 和 Outline/ 目录有有效配对文件** — 正文 ≥6000 字，大纲含 novel_tags

## 自动化流程

1. 复制 Chrome 登录态到临时目录（首次）
2. 启动 Chrome，检查番茄登录状态
3. 逐篇执行：新建短故事 → 填标题 → 填正文 → 选封面模板（最后一个）→ 点完成制作 → 检测上传成功 → 关闭封面弹窗 → 选AI → 选标签 → 点存草稿
4. 汇总报告保存到 `FanqieUploader/data/playwright-batch-latest.json`

## 单篇测试

只跑时间最新的一篇：

```bash
python -u "C:\Users\mazhihao\AppData\Local\Temp\opencode\test_single.py"
```

或直接运行校准模式（停在存草稿前）：

```bash
python -m FanqieUploader.playwright_selftest
```

## 故障处理

- **番茄跳登录页**：在弹出的 Chrome 窗口中手动登录，登录态会自动保存
- **封面弹窗不关**：脚本会检测"上传成功"提示后用 Escape 正常关闭，不暴力移除 DOM
- **标签选不上**：脚本会检测面板是否收起，自动重新打开分类下拉
- **存草稿被挡**：用 evaluate 直接点击按钮，绕过遮罩层

## 报告路径

- 批量报告：`FanqieUploader/data/playwright-batch-latest.json`
- 单篇报告：`FanqieUploader/data/test-single-latest.json`
- 校准报告：`FanqieUploader/data/playwright-selftest-latest.json`
